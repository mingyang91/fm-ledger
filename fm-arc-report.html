00 The through-line

One constraint binds the whole arc: the state-loading gap. Each round made some part of formal verification cheaper — structural cache sharing, abstract-W single proofs, direct-style's bare-value plane — but none of them moved the binding constraint. A repository fact is safe to feed the verified core as a loaded subset only if its spec is a finite point lookup. Aggregates, key-enumerations, and ANN ranks are not subset-safe, so the verified-core / unverified-shell boundary keeps landing on the cheap-CRUD / valuable-feature line.

The arc runs in two halves. Rounds 1–4 ask can Stainless verify production-shaped code cheaply at scale? — and answer yes, via a purely structural VC cache, abstract-World single proofs, and a one-line translator change. Rounds 5–6 ask does that hold on the real codebase, and is there a lower-cost production style? — and answer yes for form, but the state-loading gap is orthogonal to every technique and caps verified coverage regardless.

03 Abstract-W single-proof strictly better for reads

Instead of N monomorphic copies, emit one function abstract over W with a materialized HOF lens. The frame property proves once over abstract W; each concrete World inherits by instantiation; the per-World lens emits no contract VC.

findByIdReadOnly   postcondition  valid nativez3   one proof, reused by all Worlds
projectLens / useUnderProject       → zero contract VCs (instance cost ≈ 0; World does not re-leak)

Scope where it holds: single-repo reads, single-slice writes (setter lens + setGet), and multi-capability reads (independent per-slice conjunction). The one break is a genuine cross-slice data dependency — reading slice A from the post-state after writing slice B is INVALID over abstract W (two independent capabilities give Z3 no linking axiom). The rescue is a cross-lens commutation @law: stated abstractly as a premise, discharged only at the concrete World (O(n²) ordered slice-pairs — the only thing that monomorphizes; the op logic still proves once).

04 Service topology & pattern coverage 87 VCs, audited

A realistic 3-repo / 3-service topology, with the novel pattern being service-on-service (QuotaService → SessionService), built + independently audited (both corner-cut traps confirmed not cut).

Service-on-service translates mechanically. The abstract-spec form (depend on an opaque SessionServiceSpec[W]) is the wired-TASTy-natural one — the resolved given is emitted as a function value without opening the body. The transitive slice closure is tight and structural: QuotaService's World = {QuotaState, SessionRepoState}, with UserRepoState correctly excluded (proven by a World with no user slice verifying 15/15). No footprint inference needed.

Law-as-premise is a cheap implication (same op-proof speed whether explicit premise or trait @law); emit only the laws an op references (a 7-law bundle cost 16.4s vs 3.8s for 2 — every declared @law is verified even when unused).

Smallest achievable gap (single-repo read)

// Ox, explicit-world shape (b)
def getUser[W](id)(using h: Has[UserRepoState, W])(w: W): (W, Option[User]) = (w, h.get(w).lookup(id))
// Stainless core — same expression, modulo using↔param and the proof annotations
def getUser[W](id, users: Has[UserRepoState, W]): W => (W, Option[User]) = w => (w, users.get(w).lookup(id))
Irreducible gap: state-loading for aggregates (Ox treats the count as an opaque store primitive; the core can only express it by materializing — the regime-3 wall, plus a verification-pathological per-element invariant through Map.get); the proofs themselves (present on one side only); and concurrent writes (need a commutation argument). Gears is not on Maven Central (source-build only); from docs its using Async shape is structurally identical to Ox — neither is closer to the core.

07 Strategy verdict

single-entity CRUD + invariants
abstract-W
one proof, zero per-World VCs; folder, permission, reindex
aggregate-gated ops
structure only
logic verified, the count/limit trusted
ANN / stream / set-UPDATE
contract only
decisive computation inexpressible

For the translator: bind ops to the minimal slice (one-line emitRepoIO change), prefer abstract-W single proofs, standardize one bundling shape per slice-set, keep error ADTs minimal and identical, and don't add a monad-lowering pass. Cross-slice writes need commutation laws, discharged per concrete World — the only thing that monomorphizes.

For the production style: direct-style (Ox) makes the form-gap small and the translator mechanical-to-unnecessary — a real, lower-cost advantage over cats-effect + an IO[A] FMIO-translator, because bare values drive boundary marshalling toward zero. But it does not move the state-loading needle; that gap is orthogonal to production style and remains the ceiling on verified coverage.

Bottom line: the techniques compose into a clear strategy — abstract-W carries the operation logic for nearly everything, monomorphize-plus-cache is reduced to discharging small commutation laws, and direct-style makes reaching the core cheap. What none of them change is how much of the product the core can hold: clean for point-query CRUD, structure-only for aggregates, contract-only for ANN.
